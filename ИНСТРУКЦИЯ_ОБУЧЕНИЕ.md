# Обучение CNN для диффов

## Что делает модель

Сверточная сеть `TinyDiffCNN` получает ч/б картинку диффа размером 64×64 (после прогона Selenium и Pillow) и выдаёт класс: эталон близок (pass) или есть смысловая аномалия (fail). Это дополняет порог по проценту изменившихся пикселей, а не заменяет его полностью.

## Подготовка данных

1. Прогоните тесты на стабильных сборках и сохраняйте файлы из `shots/diffs/` там, где визуально всё ок — в `data/train/pass/`.
2. Сломанные сборки, артефакты вёрстки, «поехавшие» блоки — соответствующие диффы кладите в `data/train/fail/`.
3. Имена любые, форматы `png`, `jpg`, `webp`. Желательно не меньше десятка снимков в каждом классе.

## Команда обучения

```text
cd путь/к/нейросеть
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python train.py --data data/train --out weights/diff_cnn.pt --epochs 40 --batch 8
```

Веса появятся в `weights/diff_cnn.pt`. Путь к файлу должен совпадать с `model_path` в `config.json` (по умолчанию так и есть).

## Проверка

Запустите `python run_tests.py` с эталоном и URL. В отчёте появится строка `Model P(fail)`.

Пакетный прогон (один baseline на все строки): `python run_tests.py --batch urls.txt`. Код выхода — число упавших URL.

Макеты и стенд: см. `КАК_ЭТО_РАБОТАЕТ.md` (`storage/designs`, `manifest.json`, допуски сдвига и opening).

## Gemma через Ollama

Установите [Ollama](https://ollama.com/), выполните `ollama pull gemma3` (или другую поддерживаемую модель с тем же API). В `config.json` поля `ollama_url` и `gemma_model` должны совпадать с вашей средой. Если модель не принимает картинки, снимите галочку «Картинка в Gemma» в UI или запустите с флагом `--no-gemma-image`.

## Первый эталонный скрин

```text
python make_baseline.py --url "file:///C:/полный/путь/fixtures/demo.html"
```

Скрипт сохранит `shots/baseline.png` и пропишет `config.json`.
